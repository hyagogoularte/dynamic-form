Parabéns! Você foi aprovado para a nova etapa do nosso processo. Nela pedimos para você entregar um projeto desafio.


Desafio Front End


Você deve simular um serviço que provê uma API JSON, com um endpoint HTTP(POST) que espera a seguinte estrutura de dados para a inserção de um contato na base de dados:


{ 'token': '62bb61431348e22850828a5829c4373faafe29c1', 'secret': '51a266c2844ccd5cac83d88de88d82d05358aa51', 'lead':{ 'name':'Front end Ninja', 'email':'frontend.ninja@resultadosdigitais.com.br', 'estado':'SC', 'nível':'Ninja' } }


O serviço responde HTTP 200 para sucesso e HTTP 500 para erro.

--
Must have
1. Você deve criar um plugin para jQuery que monte um formulário através de um Hash de opções dentro de uma div HTML. Nome e E-mail são campos obrigatórios, Estado e Nível são campos extras configurados na inicialização do plugin.

2. O código deve ser publicado no Github (você pode nomear o seu plugin do jeito que achar melhor).

3. O Readme do projeto deve conter instruções sobre como instalar e usar o plugin em um site/blog em inglês.

4. O código JS deve ter testes automatizados, na RD usamos Jasmine.

5. O formulário deve ter o CSS isolado (bootstrap is cool) e deve sobreviver múltiplos navegadores (Chrome, FF, Safari) e outros softwares alternativos (IE11+).

6. O formulário deve fornecer feedback para o usuário caso haja um erro na gravação do registro.
Observações
7. Não é nescessário implementar nenhum tipo de inteligência de autenticação utilizando o Token e Secret, apenas enviar para o endpoint.


--
Usabilidade esperada
<html> ... <body> <div id="integration_form"> </div> </body> <script type="text/javascript" charset="utf-8"> $(document).ready(function() { options = { 'token':'62bb61431348e22850828a5829c4373faafe29c1', 'secret':'51a266c2844ccd5cac83d88de88d82d05358aa51', 'fields':{ 'estado':['PR','SC','SP','RS'], 'nível':['Iniciante','Intermediário','Avançado','Ninja'] } } $('#integration_form').dynaform(options); }); </script> ... </html>


--
Bônus
Através de apenas uma config, implementar o formulário em forma de Modal, chamando o plugin para criar o botão que invoca ele:
<html> ... <body> <a id="integration_form" href="#">Quero receber materiais por email</a> </body> <script type="text/javascript" charset="utf-8"> $(document).ready(function() { options = { 'token':'62bb61431348e22850828a5829c4373faafe29c1', 'secret':'51a266c2844ccd5cac83d88de88d82d05358aa51', 'modal':true, 'fields':{ 'estado':['PR','SC','SP','RS'], 'nível':['Iniciante','Intermediário','Avançado','Ninja'] } } $('#integration_form').dynaform(options); }); </script> ... </html>






Monte o formulário através do hash, botão abrindo modal é plus.
